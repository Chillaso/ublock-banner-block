# Installation Instructions

This extension is intended to be installed locally as an unpacked Manifest V3 extension.

## Option 1: Install from the existing build

Use this option when `dist/` already exists and contains a generated `manifest.json`.

1. Open Chrome or another Chromium-based browser.
2. Go to `chrome://extensions/`.
3. Enable Developer mode.
4. Click Load unpacked.
5. Select the `dist/` directory from this project.
6. Confirm that `uBlock Banner Block` appears in the extension list.
7. Pin the extension if you want quick access to the popup.

## Option 2: Build from source, then install

Use this option after cloning the repository or when you want to rebuild the extension.

1. Open a terminal in the project root.
2. Install dependencies:

```bash
npm install
```

3. Build the extension:

```bash
npm run build
```

4. Open `chrome://extensions/`.
5. Enable Developer mode.
6. Click Load unpacked.
7. Select the generated `dist/` directory.

## Verify the installation

1. Click the extension icon.
2. Confirm the popup shows the Add rule form and the Saved rules list.
3. Add a base URL and one or more CSS selectors.
4. Open a matching page and verify the targeted overlay is removed.

## Update after making changes

1. Rebuild the extension:

```bash
npm run build
```

2. Return to `chrome://extensions/`.
3. Click the Reload action for `uBlock Banner Block`.
4. Refresh any tab where you want the updated content script to run.

If you use watch mode, keep the watcher running and still reload the extension after new assets are written to `dist/`.

## Notes

- Load the `dist/` directory, not `src/`.
- The extension requests `storage` and `activeTab` permissions.
- Built-in rules come from `src/config/preconfigured-rules.yaml`.
- Custom rules are saved with `chrome.storage.sync` for the current browser profile.

## Troubleshooting

- If the browser rejects the extension, confirm that `dist/manifest.json` exists and was generated by `npm run build`.
- If the popup opens but nothing is removed from a page, verify that the page URL matches the configured base URL and that the selectors are valid.
- If changes do not appear after editing source files, rebuild and reload the extension from `chrome://extensions/`.
